Custom Roles (added on top of the built-in roles — shown in Add/Edit User "Role" dropdown)

Every custom role needs a Base Behavior — this decides what data it can actually see, reusing the same safe logic as the built-in roles:
• User (Employee) — sees only their own tickets, same as a normal employee.
• Department Role — sees all tickets raised against one department (like HR/Accounts/Management).
• Super Admin (Full Access) — full access like CEO/IT Support/Admin; pick which "Full Access" tabs it can see below. Note: since every admin action in the app checks "is this a Super Admin?", a Super-Admin-based custom role can still perform any admin action even for tabs you hide here — hiding a tab only hides it from the sidebar, it isn't a hard permission wall.
